# Headroom Sales Playbook

## Ideal Customer Profile

- AI-native engineering organizations
- Platform teams standardizing agent usage
- Teams with meaningful monthly model spend
- Security-conscious buyers that prefer self-hosting

## Qualification Questions

1. How many engineers use AI agents weekly?
2. Which providers are in use today?
3. Is there a central AI budget owner?
4. Is security review required for new infrastructure?
5. Do they need team visibility, project segmentation, or enterprise governance?

## Discovery To Close Flow

### Stage 1: Discovery
- Understand current agent stack
- Capture pain around cost, context size, and governance
- Confirm buying owner and technical champion

### Stage 2: Technical fit
- Demo proxy deployment
- Show analytics and governance path
- Map requested features to Builder, Team, Business, or Enterprise

### Stage 3: Pilot
- Set baseline metrics
- Define success criteria
- Run a time-boxed pilot

### Stage 4: Security and procurement
- Share security packet
- Answer architecture questions
- Route legal docs to counsel

### Stage 5: Close
- Present ROI
- Convert to annual contract
- Schedule onboarding

## Core Messages

- Reduce waste in agent traffic without rewriting developer workflows
- Standardize governance across providers and agent tools
- Keep the control layer inside customer infrastructure
- Give security and procurement a reviewable admin plane

## Demo Script

1. Show baseline agent request path
2. Show Headroom proxy insertion with no workflow rewrite
3. Show savings and analytics
4. Show admin auth, RBAC, audit, and retention controls
5. Show deployment paths for Docker and Kubernetes

## Objection Handling

### "Native provider caching already helps."
- Headroom is not only a caching story.
- The commercial value is cross-provider control, reversible compression, reporting, and governance.

### "We can build this ourselves."
- Most teams do not want to maintain compression logic, reporting, identity, audit, and deployment packaging across providers.

### "We need security review."
- Use the existing packet and enterprise controls as the answer path.

## Pilot Success Metrics

- first request through the proxy
- measurable token savings
- working admin auth path
- at least one report exported
- at least one stakeholder review meeting completed

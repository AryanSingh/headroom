# Headroom Pilot Offer Template

## Purpose

Use this template to structure a paid or design-partner pilot without improvising the commercial terms.

## Scope

The pilot includes:
- Headroom deployment in the customer environment
- Baseline and end-of-pilot savings review
- Admin plane setup appropriate to tier
- Weekly check-ins

The pilot does not include:
- Unlimited custom engineering
- Production SLA unless separately agreed
- Formal compliance certification deliverables

## Success Criteria

Use 3 to 5 measurable criteria:

1. Compression or savings target
2. Quality preservation target
3. Admin or governance milestone
4. Deployment stability milestone
5. Renewal or conversion decision date

## Example Success Criteria

- At least 50% token savings on targeted traffic classes
- No material quality regression reported by pilot users
- First exportable savings report delivered by day 7
- Enterprise admin auth and audit flow verified by day 10

## Roles

### Customer
- provides environment access
- provides technical owner
- participates in weekly review

### Headroom
- provides onboarding support
- provides weekly metric review
- provides conversion recommendation

## End Of Pilot Decision

At pilot close, both sides choose one:

- convert to annual paid contract
- extend pilot once with a defined reason
- close pilot with written lessons learned

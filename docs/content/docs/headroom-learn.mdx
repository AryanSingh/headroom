title: cutctx learn
description: Agent self-improvement — mines failed sessions, extracts lessons, writes corrections to CLAUDE.md or AGENTS.md.
---

`cutctx learn` watches your AI agent's sessions, finds the failures, and writes the fixes into `CLAUDE.md` or `AGENTS.md` so every future session starts smarter.

## The Problem It Solves

Every AI coding agent session produces failures: wrong tool calls, misread file structures, incorrect assumptions about your codebase. You notice, you fix it manually, you move on. Next session — the agent makes the same mistake again. The agent has no memory of what didn't work.

`cutctx learn` closes that loop automatically.

## How It Works

```bash
cutctx learn [--session-dir ~/.headroom/sessions] [--output CLAUDE.md]
```

1. **Mines session logs** — reads compressed session records from the CutCtx proxy (every request/response pair is stored locally)
2. **Detects failure patterns** — looks for: repeated tool errors, retry loops, contradicted assumptions, explicit corrections from the user, and assistant backtracking
3. **Extracts corrections** — uses an LLM call (configurable model) to summarize what went wrong and what the correct behavior should be
4. **Writes to `CLAUDE.md` / `AGENTS.md`** — appends structured corrections in the format your agent reads on every session start

The output is human-readable and diff-friendly. You review and commit it. The agent reads it on its next session start.

## Example

After a session where Claude Code incorrectly tried to run tests with `pytest` (your project uses `cargo test`):

```bash
$ cutctx learn --last 5

Analyzing 5 sessions (847 tool calls)...

Found 3 correction candidates:

1. [HIGH CONFIDENCE] Test runner assumption
   Pattern: `pytest` called 4 times across 2 sessions, always failed
   Correction: This project uses `cargo test` not `pytest`. Python tests do not exist.

2. [MEDIUM] Import resolution
   Pattern: Agent repeatedly tried `from headroom.core import X` — module is `headroom_core`
   Correction: The Rust core is imported as `import headroom_core` (underscore), not `headroom.core`

3. [LOW] File path assumption
   Pattern: Agent looked for config in `~/.config/headroom/` — not found, then found in `~/.headroom/`
   Correction: CutCtx config is at ~/.headroom/, not ~/.config/headroom/

Write these 3 corrections to CLAUDE.md? [Y/n]
```

After confirmation, `CLAUDE.md` gets:

```markdown
## Corrections (auto-generated by cutctx learn, 2026-06-15)

- Use `cargo test` to run tests. Do not use `pytest` — there are no Python tests in this repo.
- Import the Rust core as `import headroom_core` (underscore). `from headroom.core import X` does not work.
- CutCtx config is at `~/.headroom/`. Do not look in `~/.config/headroom/`.
```

## Configuration

```bash
# Run on last N sessions (default: all)
cutctx learn --last 10

# Point to a specific CLAUDE.md
cutctx learn --output /path/to/CLAUDE.md

# Use a different model for pattern extraction
cutctx learn --model claude-haiku-4-5  # default: cheapest available

# Dry run — show corrections without writing
cutctx learn --dry-run

# Auto-confirm without prompting (for CI/scripted use)
cutctx learn --yes
```

## Scheduling

Run automatically after every session by adding a post-hook:

```bash
# In your shell profile or .envrc
cutctx wrap claude --post-hook "cutctx learn --last 1 --yes"
```

Or schedule a weekly digest:

```bash
# crontab: every Monday at 9am
0 9 * * 1 cutctx learn --last 50 --output ~/projects/myrepo/CLAUDE.md --yes
```

## Supported Agent Files

| Agent | File written |
|---|---|
| Claude Code | `CLAUDE.md` |
| OpenAI Codex | `AGENTS.md` |
| Cursor | `.cursorrules` |
| Aider | `.aider.conf.yml` (instructions section) |
| Custom | `--output <any file>` |

## Privacy

`cutctx learn` runs entirely locally. Session logs are stored in `~/.headroom/sessions/` on your machine. The only external call is the LLM inference step for pattern extraction — this can be directed at any provider, including a local Ollama instance:

```bash
cutctx learn --model ollama/llama3  # fully offline
```

No session data is sent to CutCtx servers.

## Why This Matters

Most agents improve by training on feedback from millions of users. `cutctx learn` gives your agent improvement from your specific codebase, your specific patterns, and your specific failures — without sharing anything.

Teams that use `cutctx learn` + `CLAUDE.md` commit corrections as part of normal development workflow. The agent gets measurably better at your repo over time. New team members get those corrections automatically when they clone and start their first session.

## Getting Started

```bash
pip install "cutctx-ai[all]"
cutctx wrap claude    # start a session with learning enabled
# ... do some work ...
cutctx learn          # mine the session and update CLAUDE.md
```

The first run typically finds 3–8 corrections. After a month of daily use, most teams see near-zero repeat failures on issues that have been corrected before.

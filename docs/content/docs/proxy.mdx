---
title: Proxy Server
description: Run the Cutctx proxy to compress LLM traffic for any client — Claude Code, Cursor, OpenAI SDK, or custom apps.
---

The Cutctx proxy is a standalone HTTP server that compresses all LLM traffic passing through it. Point any client at the proxy and get automatic context optimization.

## Starting the proxy

```bash
# Basic usage
cutctx proxy

# Custom host and port
cutctx proxy --host 0.0.0.0 --port 8080

# With logging and budget
cutctx proxy \
  --log-file /var/log/cutctx.jsonl \
  --budget 100.0
```

Telemetry is enabled by default. Opt out with `CUTCTX_TELEMETRY=off` or `--no-telemetry`.

## CLI options

### Core

| Option | Default | Description |
|--------|---------|-------------|
| `--host` | `127.0.0.1` | Host to bind to |
| `--port` | `8787` | Port to bind to |
| `--workers` | `1` | Number of Uvicorn worker processes |
| `--limit-concurrency` | `1000` | Maximum concurrent connections before Uvicorn returns 503 |
| `--max-connections` | `500` | Maximum upstream HTTP connections |
| `--max-keepalive` | `100` | Maximum upstream keep-alive connections |
| `--mode` | `token` | Optimization mode: `token` prioritizes compression, `cache` preserves provider prefix-cache stability |
| `--no-optimize` | `false` | Disable optimization (passthrough mode) |
| `--compression-mode` | `safe` | `off` preserves request bytes, `safe` uses conservative routing, `aggressive` applies tighter compression to plain prose |
| `--deterministic` | `false` | Disable the ML compressor and keep rule-based deterministic compression only |
| `--no-cache` | `false` | Disable semantic caching |
| `--no-rate-limit` | `false` | Disable rate limiting |
| `--log-file` | None | Path to JSONL log file |
| `--log-messages` | `false` | Store full request/response content for the live feed |
| `--budget` | None | Daily budget limit in USD |
| `--openai-api-url` | `https://api.openai.com` | Custom OpenAI API URL |
| `--anthropic-api-url` | Anthropic default | Custom Anthropic API URL |
| `--gemini-api-url` | Gemini default | Custom Gemini API URL |
| `--backend` | `anthropic` | Backend: `anthropic`, `bedrock`, `openrouter`, `anyllm`, or `litellm-<provider>` |
| `--no-telemetry` | `false` | Disable anonymous telemetry |
| `--stateless` | `false` | Disable filesystem writes and keep runtime state in memory |

### Context management

| Option | Default | Description |
|--------|---------|-------------|
| `--mode token` | `token` | Prioritize token compression. This is the default. |
| `--mode cache` | `token` | Preserve prior turns to maximize provider prefix-cache hit rate. |
| `--intercept-tool-results` | `false` | Opt into tool-result interceptors such as ast-grep Read outlining. |
| `--no-read-lifecycle` | `false` | Disable stale/superseded Read-output compression. |
| `--code-aware` / `--no-code-aware` | disabled | Enable or disable AST-based code compression. Requires `cutctx-ai[code]`. |
| `--code-graph` | `false` | Index the current project and watch files via codebase-memory-mcp. |

By default, the proxy uses the shared **ContentRouter** pipeline. It routes text, logs, JSON, code, images, and tool outputs through the currently enabled compressors and preserves reversible CCR markers where applicable.

### Compression policy

Use `safe` for the default, low-latency behaviour. It uses deterministic prose
compression for ordinary text and keeps specialised code, JSON, log, diff, and
search routes. `aggressive` is an explicit quality/ratio opt-in that asks the
Kompress ML compressor for a 40% target ratio on prose; it keeps the same
format-aware safeguards for specialised content. If the optional ML runtime is
not installed or declines to reduce a prose block, aggressive mode falls back
to a deterministic sentence selector that keeps headings, identifiers,
numerical constraints, and a representative sentence. `off` bypasses the
complete router stage, including normalization and lifecycle filtering, and
forwards the original request bytes unchanged.

Kompress trades latency for learned compression quality. It is selected only
by `aggressive` mode or an explicit per-request Kompress option, never by the
default safe route. This keeps latency-sensitive interactive sessions on the
deterministic path while preserving the higher-compression option when its
tradeoff is worthwhile.

### Output verbosity steering

The proxy adds a short, guarded terse-output instruction for search, list, and
summary requests. It never applies that instruction to code-edit or debugging
tasks, and it does not set or lower `max_tokens`; explicit client output limits
always remain authoritative.

```bash
# Quality-first default
cutctx proxy --compression-mode safe

# Tighter prose compression when context budget matters most
cutctx proxy --compression-mode aggressive

# Audit or incident mode: no router-stage mutation
cutctx proxy --compression-mode off
```

```bash
# Maximize compression
cutctx proxy --mode token

# Preserve provider prefix cache stability
cutctx proxy --mode cache
```

### Optional features

| Option | Default | Description |
|--------|---------|-------------|
| `--memory` | `false` | Enable persistent user memory and provider-appropriate memory tools |
| `--memory-db-path` | `{cwd}/.cutctx/memory.db` | Override the memory SQLite path |
| `--no-memory-tools` | `false` | Disable automatic memory tool injection |
| `--no-memory-context` | `false` | Disable automatic memory context injection |
| `--memory-top-k` | `10` | Number of memories to inject as context |
| `--learn` | `false` | Enable live traffic learning; implies `--memory` |
| `--no-learn` | `false` | Explicitly disable traffic learning |
| `--min-evidence` | `5` | Minimum observations before a learned pattern is persisted |
| `--codex-wire-debug` | `false` | Write local Codex wire snapshots and matching proxy log traces |

```bash
cutctx proxy --memory
cutctx proxy --learn --min-evidence 3
cutctx proxy --codex-wire-debug
```

When memory tools are injected into a session, Cutctx keeps their serialized bytes
stable for the rest of that session so prefix caches do not churn. The behavior is
controlled by:

- `CUTCTX_TOOL_INJECTION_STICKY` — `enabled` by default; set to `disabled` only for
  explicit rollback or diagnostics.
- `CUTCTX_TOOL_TRACKER_MAX_SESSIONS` — maximum sticky sessions kept in the bounded
  LRU tracker.
- `CUTCTX_TOOL_TRACKER_PATH` — optional JSON persistence file for sticky session
  state.

## Dashboard surface

The production dashboard is served at `/dashboard` and also accepts the `/admin`
basename when the React app is mounted behind that legacy path. The live view
refreshes proxy stats every 5 seconds and historical rollups every 60 seconds.
The request feed panel refreshes on its own 5-second cadence when opened.

The dashboard build also carries a commented-out production obfuscation hook in
`dashboard/vite.config.js`. Source maps are explicitly disabled, but the
obfuscator plugin is not active in the current build.

### Enterprise release markers

Enterprise releases may carry a signed marker manifest that associates a build
with its license metadata. The proxy runtime only parses that pre-existing
metadata and can verify explicitly supplied license IDs against a local
database. It does not rewrite package source files or scan compiled binaries;
those release-engineering and forensic operations are intentionally outside the
runtime distribution.

<Callout type="info" title="LLMLingua remains optional">
LLMLingua is still available as an optional compression path and is reported in the proxy's live capability status. Cutctx's default proxy compression path uses ContentRouter plus the current built-in compressors, including Kompress where applicable.
</Callout>

## API endpoints

### `GET /health`

```bash
curl http://localhost:8787/health
```

```json
{
  "service": "cutctx-proxy",
  "status": "healthy",
  "ready": true,
  "alive": true
}
```

This public probe intentionally omits provider URLs, raw connection errors,
and configuration. Operators with admin authentication can use
`GET /health/config` for those diagnostics.

### `GET /stats`

Live session statistics plus durable `persistent_savings` totals. Stored at `~/.cutctx/proxy_savings.json` (override with `CUTCTX_SAVINGS_PATH`). Savings persistence is journaled asynchronously for request-path performance; graceful shutdown flushes it. See [Metrics & Monitoring](metrics#savings-durability) for the bounded hard-kill data-loss window.

```bash
curl http://localhost:8787/stats
```

### `GET /stats-history`

Durable history with hourly, daily, weekly, and monthly rollups. Powers the `/dashboard` view.

```bash
curl http://localhost:8787/stats-history
curl "http://localhost:8787/stats-history?format=csv&series=weekly"
```

### `GET /transformations/traces/{request_id}`

Returns the operator trace for a request, including an additive versioned
`decision_receipt`. This local-admin endpoint is the source of the dashboard’s
request inspector: it explains routing retention or changes, compression,
cache observations, CCR references, savings attribution, and what evidence was
not available.

```bash
curl -H "x-cutctx-admin-key: $CUTCTX_ADMIN_API_KEY" \
  http://localhost:8787/transformations/traces/req_123
```

```json
{
  "trace": {
    "request_id": "req_123",
    "decision_receipt": {
      "schema_version": 1,
      "observation": {
        "completeness": "complete",
        "missing": [],
        "payload_capture": "disabled"
      },
      "routing": {
        "status": "retained",
        "reason": "workload_not_downgradeable",
        "requested_model": "gpt-5.4",
        "effective_model": "gpt-5.4"
      },
      "cache": {
        "provider_prompt_cache": {"status": "unobserved", "read_tokens": 0}
      },
      "ccr": {"status": "not_used", "references": []}
    }
  }
}
```

Decision receipts remain useful with `--log-messages` disabled (the default):
they contain metadata and never copy prompts, responses, tool results,
retrieved CCR content, credentials, or raw configuration values. A cache state
of `unobserved` means Cutctx did not receive enough provider telemetry; it is
not a cache miss. Provider prompt-cache savings are reported as observed
provider savings, not as savings created by Cutctx. CCR references contain
only opaque hashes and availability metadata, never retrieved content.

Older request-log rows are returned with
`observation.completeness="legacy"`; their receipt is inferred only from the
historical fields that were stored. Treat missing legacy evidence as unknown.

### `POST /v1/messages`

Anthropic API format. The proxy compresses messages, forwards to Anthropic, and returns the response.

### `POST /v1/chat/completions`

OpenAI API format. The proxy compresses messages, forwards to OpenAI, and returns the response.

### `POST /v1/responses`

OpenAI Responses API format. The proxy compresses `input` payloads where applicable, forwards the request, and returns the response.

For Codex-compatible clients, the proxy also accepts these alias paths and routes them through the same handler:
- `POST /v1/codex/responses`
- `POST /backend-api/responses`
- `POST /backend-api/codex/responses`

Matching WebSocket and subpath aliases are also supported for Codex flows.

### `POST /v1internal:streamGenerateContent`

Google Cloud Code Assist / Antigravity compatibility endpoint used by Pi-style `google-gemini-cli` and `google-antigravity` providers.

The proxy also accepts:
- `POST /v1/v1internal:streamGenerateContent`

### `POST /v1/compress`

Compression-only endpoint. Compresses messages without calling any LLM. Used by the TypeScript SDK.

**Request:**
```json
{
  "messages": [{ "role": "user", "content": "..." }],
  "model": "gpt-4o"
}
```

**Response:**
```json
{
  "messages": [{ "role": "user", "content": "..." }],
  "tokens_before": 15000,
  "tokens_after": 3500,
  "tokens_saved": 11500,
  "compression_ratio": 0.23,
  "transforms_applied": ["router:smart_crusher:0.35"],
  "ccr_hashes": ["a1b2c3"]
}
```

Set `x-cutctx-bypass: true` to skip compression.

## Agent wrapping

Use `cutctx wrap` to transparently proxy any CLI tool:

```bash
# Claude Code
cutctx wrap claude

# OpenAI Codex
cutctx wrap codex

# Gemini CLI
cutctx wrap gemini

# Aider
cutctx wrap aider

# Cursor
cutctx wrap cursor
```

Or set the base URL manually:

```bash
# Claude Code
ANTHROPIC_BASE_URL=http://localhost:8787 claude

# Cursor / any OpenAI-compatible client
OPENAI_BASE_URL=http://localhost:8787/v1 cursor
```

## Production deployment

### gunicorn

```bash
pip install gunicorn

gunicorn cutctx.proxy.server:app \
  --workers 4 \
  --bind 0.0.0.0:8787 \
  --worker-class uvicorn.workers.UvicornWorker
```

### Docker

```dockerfile
FROM python:3.11-slim
RUN apt-get update && apt-get install -y --no-install-recommends build-essential \
    && pip install "cutctx-ai[proxy]" \
    && apt-get purge -y build-essential && apt-get autoremove -y \
    && rm -rf /var/lib/apt/lists/*
EXPOSE 8787
CMD ["cutctx", "proxy", "--host", "0.0.0.0"]
```

<Callout type="info" title="Build dependencies">
`build-essential` is required at install time because `cutctx-ai` includes `hnswlib`, a C++ extension compiled from source. It is removed after installation to keep the image slim.
</Callout>

How It Works

Three steps. Zero configuration.

1

Mine

Scans your agent's session logs (Claude Code, Codex, Cursor, Aider) for patterns of repeated mistakes.

2

Analyze

Groups failures by type, counts occurrences, and identifies the root cause of each recurring issue.

3

Write

Auto-generates corrections in CLAUDE.md so your agent never makes the same mistake again.

# Before: agent made the same mistake 4 times
$ cat CLAUDE.md
# (empty or minimal)

$ cutctx learn --last 5
Scanning 5 sessions...
Found 3 repeated failure patterns:
1. Used pytest instead of cargo test (4 occurrences)
2. Wrong import path for headroom.proxy (3 occurrences)
3. Tried editing .cargo/config.toml directly (2 occurrences)
Writing corrections to CLAUDE.md...

# After: CLAUDE.md now contains 3 auto-corrections
$ cat CLAUDE.md
# CLAUDE.md now contains 3 auto-corrections

## Testing
Always use `cargo test --release -p headroom-core`
for Rust tests. Never use pytest for Rust code.

## Imports
Use `from headroom.proxy.server import create_app`
not `from headroom import server`.

## Configuration
Edit Rust config via CLI flags, not
direct .cargo/config.toml edits.
